# Copilot Studio Tool Readiness — Candidate Evaluation Council API

Status: **E4 registration readiness documentation and curated contract only.**
No Copilot Studio tool has been registered by this repository, no Entra app
registration exists for this lab, and no portal configuration is automated
here. This document records what the implemented API and the E4 curated
registration artifact provide for human-led tool registration.

## What the implemented contract already provides

The committed source API contract is `openapi/evaluations-api.json`, generated
from the live app factory by `scripts/export_openapi.py` (drift-checked by
test). It remains OpenAPI 3.1 and is not replaced by the Copilot registration
artifact.

The E4 curated registration artifact is
`openapi/copilot-studio/evaluations-tool.swagger.json`, generated by
`scripts/export_copilot_openapi.py`. It is Swagger 2.0 for Copilot Studio /
Power Platform custom connector compatibility and intentionally exposes only
the two registration actions. Its request schema is fixture-reference-only for
E4: it does not expose inline resume or cover-letter text fields, and it must
not be used for real applicant data.

| Readiness item | Status in the implemented API |
|---|---|
| Stable operation IDs | `submitEvaluation` (`POST /api/evaluations`), `getEvaluation` (`GET /api/evaluations/{evaluation_id}`) |
| Tool-suitable descriptions | Operation `summary`/`description` state purpose, advisory-only boundary, status vocabulary, and HTTP mapping in business language |
| Curated Swagger 2.0 registration artifact | `openapi/copilot-studio/evaluations-tool.swagger.json` with placeholder host `function-app-host.example`, `basePath: /`, `https` only, and exactly two actions |
| Fixture-reference-only request shape | The curated Copilot artifact exposes `position_id`, `candidate_ref`, optional `idempotency_key`, optional `evaluation_question`, and optional `requested_rigor`; inline applicant text remains outside E4 Copilot registration scope |
| Function key auth representation | The curated Swagger declares a header API key security definition named `x-functions-key`; the key value is supplied only through secure Copilot Studio / Power Platform connection settings |
| Temporary lab identity headers | The curated Swagger exposes `X-Lab-Actor-Id`, `X-Lab-Roles`, and optional `X-Lab-Actor-Display` as visible/testable header parameters; these are simulated lab auth only, not Entra identity |
| Standard response envelope | Adopted envelope subset: `status`, `evaluation_id`, `case_id` (always null — case-less), `correlation_id`, `user_message`, `safe_details`, `result`, `errors`, `warnings` |
| Fixed status vocabulary | Emitted: `completed | blocked | validation_failed | unauthorized`; declared-reserved: `needs_input`, `error`. Business outcomes return HTTP 200; malformed body 400; missing identity 401; wrong role 403 |
| Correlation ID header | `X-Correlation-Id` accepted on requests (POST and GET) and returned on responses that have a correlation id to carry: the server-assigned id (also in the envelope) takes precedence; a caller-supplied value is echoed back when no envelope correlation id exists. Early-failure responses (400/401/403, envelope-less `validation_failed`) carry the header only if the caller sent one |
| Idempotency key header | `Idempotency-Key` accepted on `POST` as an equivalent of the body field `idempotency_key` (one required; mismatch → HTTP 400); replays return the original result without re-running the council |
| No backend selection in requests | The request schema contains **no** provider/model/deployment/endpoint/agent field; provider selection is server-side configuration only |
| Advisory-only output | Every result carries `decision_support_only: true` and `human_review_required: true`; recommendation labels are the fixed advisory enum; `effective_rigor` and the escalation-policy outcome are in the result |
| Synthetic examples only | All fixtures and documented examples are synthetic (`cand-sample-001`, `pos-sample-001`) |

## Manual registration guide

Use
[`copilot-studio/registration-guide.md`](./copilot-studio/registration-guide.md)
for the human portal checklist and hosted smoke commands. The guide includes
both paths for the temporary simulated lab identity headers:

- Preferred E4 lab path: configure static/default/internal `X-Lab-*` headers
  if the Copilot Studio or connector UI supports them.
- Fallback path: import and review the two actions, but do not claim
  end-to-end Copilot invocation until connector policy/header support is
  configured or a later Entra-auth slice replaces simulated headers.

## Tool description drafts (for the future registration)

- **submitEvaluation** — "Submit one synthetic candidate evaluation to the
  advisory Calibrated Evaluation Council for a sample position and approved
  rubric. Returns an advisory, evidence-grounded result that always requires
  human review. Never makes or implies a hiring decision."
- **getEvaluation** — "Retrieve the full persisted audit record of a prior
  evaluation by evaluation_id, including every council role output, rigor
  resolution, escalation triggers, and quality-gate results. Advisory decision
  support only."
